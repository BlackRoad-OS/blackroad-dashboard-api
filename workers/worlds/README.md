## 1. Overview

The Worlds worker fetches and serves AI world generation data from the `BlackRoad-OS-Inc/blackroad-agents` GitHub repository. It parses world artifact filenames to extract metadata (title, type, node, timestamp) and provides real-time stats on the 125,000+ world artifacts generated by the Pi fleet.

## 3. World Artifact Schema

World artifacts are stored in the `blackroad-agents` repo as Markdown files with the naming convention:

```
<node-dir>/<YYYYMMDD>_<HHMMSS>_<type>_<slug>.md
```

Parsed fields:

| Field | Description |
|---|---|
| `id` | Full filename (without `.md`) |
| `title` | Human-readable title (slug → Title Case) |
| `node` | Pi node that generated the world |
| `type` | Artifact type: `world`, `lore`, or `code` |
| `path` | Full path in the GitHub repo |
| `timestamp` | ISO 8601 generation timestamp |
| `link` | Direct GitHub URL to the artifact |

|---|---|
| `GITHUB_TOKEN` | GitHub API token for reading `blackroad-agents` (increases rate limit from 60 to 5,000 req/hr) |

```bash
wrangler secret put GITHUB_TOKEN --name worlds-blackroadio
```

## 6. Deployment

```bash
cd workers/worlds
wrangler deploy
```

Route: `worlds.blackroad.io/*` (zone: `blackroad.io`)

Automated via [`.github/workflows/deploy-worlds.yml`](../../.github/workflows/deploy-worlds.yml) on push to `workers/worlds/**`.
